Section 1. SQL 기본 (10개 주제)

01SQL 연산순서

작성 순서와 실행 순서는 다르다. 논리적 처리 흐름을 반드시 외운다.

  • 실행 순서: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
  • SELECT에서 정의한 별칭은 WHERE / HAVING에서 사용 불가 (아직 생성 전)
  • ORDER BY는 실행 순서 마지막이므로 SELECT 별칭 사용 가능
  • FROM 단계에서 테이블 결합(JOIN), WHERE 단계에서 행 필터링이 이루어짐
  • GROUP BY 이후에는 개별 행이 아닌 그룹 단위로 처리됨
  • 서브쿼리가 있으면 서브쿼리 내부가 먼저 실행된 뒤 외부 쿼리가 실행됨
SELECT department, COUNT(*) AS cnt -- ③ SELECT FROM employees -- ① FROM WHERE salary > 3000 -- ② WHERE GROUP BY department -- ④ GROUP BY HAVING COUNT(*) >= 2 -- ⑤ HAVING ORDER BY cnt DESC; -- ⑥ ORDER BY (별칭 사용 가능)
02DISTINCT

중복 행 제거. NULL도 하나의 값으로 취급. 여러 컬럼 조합 기준으로 동작.

  • SELECT DISTINCT col → col 값 기준 중복 제거
  • SELECT DISTINCT col1, col2 → (col1, col2) 조합 기준 중복 제거
  • NULL이 여러 행에 있어도 DISTINCT 후 NULL은 1개만 남음
  • COUNT(DISTINCT col) : NULL을 제외하고 중복 제거 후 카운트
  • DISTINCT는 SELECT 키워드 바로 뒤 한 번만 쓸 수 있음
  • ORDER BY나 집계 함수와 함께 자주 출제됨 → 결과 행 수를 직접 계산하는 연습 필요
03Alias (별칭)

컬럼·테이블에 임시 이름 부여. 사용 가능 위치와 따옴표 규칙을 함께 본다.

  • 컬럼 별칭: col AS 별칭 또는 col 별칭 (AS 생략 가능)
  • 별칭에 공백·특수문자·예약어가 포함되면 큰따옴표(" ") 필수 (오라클 표준)
  • 테이블 별칭: FROM employees e → 이후 원본명 employees 사용 불가
  • 컬럼 별칭은 ORDER BY에서 사용 가능, WHERE·GROUP BY에서는 원칙적으로 불가
  • 일부 DBMS(MySQL 등)는 GROUP BY에서 SELECT 별칭 허용하지만 시험은 표준 기준으로 판단
SELECT salary * 12 AS "연봉(원)", -- 공백 포함 → 큰따옴표 department dept_name -- AS 생략 가능 FROM employees e -- 테이블 별칭 WHERE e.salary > 5000 -- 원본명 대신 별칭 사용 ORDER BY "연봉(원)" DESC; -- ORDER BY에서 별칭 사용 가능
04CONCAT

문자열 연결. DBMS마다 문법이 다르며 NULL 처리 동작도 주의.

  • 오라클: || 연산자 또는 CONCAT(a, b) — CONCAT은 인수 2개만
  • SQL Server: + 연산자 또는 CONCAT(a, b, ...) — 다중 인수 가능
  • 오라클 CONCAT('A', NULL) = 'A' — NULL을 빈 문자열처럼 처리
  • SQL Server에서 'A' + NULL = NULL — NULL 전파 주의
  • 세 문자열 연결 시 오라클: CONCAT(CONCAT(a, b), c) 또는 a || b || c
  • 표준 SQL은 || 연산자가 공식 연결 연산자
-- 오라클 SELECT CONCAT(first_name, last_name) AS name1, -- 2개만 first_name || ' ' || last_name AS name2 -- || 다중 가능 FROM employees;
05논리연산자

NOT > AND > OR 우선순위. 괄호가 최우선. 조건 누락 문제의 핵심.

  • 우선순위: NOT > AND > OR
  • 괄호가 있으면 괄호 내부가 최우선 — 괄호 없이 AND/OR 혼용 시 의도와 결과 다를 수 있음
  • A OR B AND C = A OR (B AND C) — AND가 먼저 묶임
  • NULL과의 논리 연산: NULL AND TRUE = NULL, NULL OR TRUE = TRUE, NOT NULL = NULL
  • 3값 논리(TRUE / FALSE / UNKNOWN): NULL 포함 비교는 UNKNOWN 반환 → WHERE 조건 불통과
-- 주의: 아래 두 쿼리는 결과가 다름 WHERE dept = 'A' OR dept = 'B' AND salary > 5000 -- B이고 급여>5000, 또는 A WHERE (dept = 'A' OR dept = 'B') AND salary > 5000 -- A또는B이고 급여>5000
06비교연산자 / IN / BETWEEN / LIKE / EXISTS / ANY / ALL

조건절의 모든 연산자를 한 번에 정리. 서브쿼리와의 연결이 핵심.

  • BETWEEN a AND b : a 이상 b 이하 (양쪽 포함)
  • IN (v1, v2, ...) : 목록 중 하나라도 일치. IN(서브쿼리) 형태도 빈출
  • LIKE : % (0개 이상 임의 문자), _ (정확히 1개 문자)
  • EXISTS : 서브쿼리 결과가 1건이라도 있으면 TRUE. 행 존재 여부만 봄
  • > ANY : 서브쿼리 결과 중 하나라도 크면 TRUE (= > MIN)
  • > ALL : 서브쿼리 결과 모두보다 크면 TRUE (= > MAX)
  • NULL 비교: = NULL, <> NULL 은 항상 UNKNOWN → IS NULL 사용
  • NOT IN에 NULL이 포함되면 전체 결과 0건 주의

col > ALL (서브쿼리)서브쿼리 최댓값보다 크면 TRUE= > MAX(...)

07ESCAPE

LIKE에서 %, _ 자체를 검색할 때 사용. 이스케이프 문자는 사용자가 직접 지정.

  • LIKE 'A\_%' ESCAPE '\' → 'A_'로 시작하는 문자열 검색
  • ESCAPE 문자는 바로 뒤 한 글자를 일반 문자로 취급
  • ESCAPE 문자를 \ 외에 다른 문자(@ 등)로 지정해도 무방
  • ESCAPE는 LIKE와 함께 쓸 때만 의미 있음 — 단독 사용 불가
  • 예: 'A%B' 검색 → LIKE 'A\%B' ESCAPE '\'
-- % 기호가 실제로 들어간 행 검색 SELECT * FROM products WHERE description LIKE '%30\%%' ESCAPE '\'; -- → 문자열 어딘가에 '30%' 포함
08TOP / ROWNUM / FETCH

상위 N건 추출. DBMS별 문법과 정렬 전후 처리 순서가 핵심.

  • ROWNUM (Oracle): FROM 처리 직후, ORDER BY 전 번호 부여 → 정렬 후 상위 N건 원하면 인라인 뷰 필수
  • TOP N (SQL Server): ORDER BY 적용 후 N건 반환. WITH TIES로 동점 포함 가능
  • FETCH FIRST N ROWS ONLY (표준 SQL): 오라클 12c+, PostgreSQL 등 지원
  • ROWNUM = 1은 가능, ROWNUM = 2 는 결과 없음 (ROWNUM은 1부터 순차 부여)
  • ROWNUM <= N 방식으로 상위 N건 추출
-- 오라클: 정렬 후 상위 3건 (인라인 뷰 필수) SELECT * FROM ( SELECT * FROM employees ORDER BY salary DESC ) WHERE ROWNUM <= 3; -- 표준 SQL (오라클 12c+) SELECT * FROM employees ORDER BY salary DESC FETCH FIRST 3 ROWS ONLY;
09NULL / NULL 처리 함수

NULL은 모른다(Unknown). 비교·연산·집계 모두 특수하게 동작한다.

  • NULL 비교: = NULL, <> NULL 모두 UNKNOWN → 반드시 IS NULL / IS NOT NULL 사용
  • NULL과의 산술 연산 결과는 항상 NULL (10 + NULL = NULL)
  • NVL(col, 대체값): 오라클 전용. col이 NULL이면 대체값 반환
  • ISNULL(col, 대체값): SQL Server 전용
  • COALESCE(a, b, c, ...): 표준 SQL. 첫 번째 non-NULL 반환
  • NULLIF(a, b): a = b 이면 NULL 반환, 다르면 a 반환 (0 나누기 방지에 활용)
  • NVL2(col, not_null_val, null_val): col이 NULL이 아닐 때·NULL일 때 각각 반환 (오라클)
  • 집계 함수는 NULL을 제외 — AVG는 NULL 제외 후 나눔 (0으로 나누기 위험)

10ORDER BY

정렬 기본값 ASC. NULL 위치, 별칭·위치번호 사용 규칙을 함께 외운다.

  • 기본 정렬: ASC (오름차순) 생략 가능. DESC는 내림차순
  • 오라클 ASC: NULL을 최댓값(마지막)으로 취급
  • SQL Server ASC: NULL을 최솟값(처음)으로 취급
  • 명시적 제어: NULLS FIRST / NULLS LAST (오라클 지원)
  • ORDER BY에 컬럼명, SELECT 순서 위치번호, SELECT 별칭 모두 사용 가능
  • 다중 정렬: ORDER BY col1 ASC, col2 DESC
  • ORDER BY는 항상 마지막에 실행되므로 결과 집합 전체에 적용됨
SELECT name, salary, department FROM employees ORDER BY 3 ASC, -- 3번째 컬럼(department) 오름차순 salary DESC -- 같은 부서 내 급여 내림차순 NULLS LAST; -- NULL은 맨 뒤로

11숫자 함수

ROUND/TRUNC/CEIL/FLOOR의 반환값 차이를 직접 계산해보며 외운다.

  • ROUND(n, d): d 자리에서 반올림. d 생략 시 정수 반올림
  • TRUNC(n, d): d 자리에서 버림 (오라클). 음수 d는 십/백 단위
  • CEIL(n): n보다 크거나 같은 가장 작은 정수 (올림)
  • FLOOR(n): n보다 작거나 같은 가장 큰 정수 (내림)
  • 음수에서: CEIL(-1.5) = -1, FLOOR(-1.5) = -2 주의
  • MOD(n, m): n을 m으로 나눈 나머지. MOD(10, 3) = 1
  • POWER(n, m): n의 m제곱. SQRT(n): 제곱근
  • ROUND(123.456, -2) = 100 — 음수 자리수는 정수 자리 반올림

12문자 함수

SUBSTR 시작 위치·길이, TRIM 범위, LPAD 자리수를 인수별로 정확히 외운다.

  • UPPER(s) / LOWER(s): 대/소문자 변환
  • SUBSTR(s, 시작, 길이): 시작 위치는 1부터. 길이 생략 시 끝까지. 음수 시작은 뒤에서 셈
  • LENGTH(s): 문자 수. LENGTHB(s): 바이트 수 (한글 차이)
  • INSTR(s, 찾을문자, 시작, 회차): 위치 반환, 없으면 0
  • LPAD(s, n, 채울문자): 왼쪽 채우기. RPAD: 오른쪽 채우기
  • TRIM(s): 양쪽 공백 제거. LTRIM / RTRIM: 한쪽만
  • TRIM(LEADING '0' FROM '00123') = '123' — 특정 문자 제거 가능
  • REPLACE(s, 찾을, 바꿀): 문자열 치환. 바꿀 생략 시 제거
  • TRANSLATE(s, from, to): 문자 단위 1:1 치환
SELECT SUBSTR('ABCDEF', 2, 3) -- 'BCD' (2번째부터 3자) ,SUBSTR('ABCDEF', -2) -- 'EF' (뒤에서 2번째부터) ,LPAD('7', 4, '0') -- '0007' ,INSTR('ABCABC','B',1,2) -- 5 (1번째부터 탐색, 2번째 등장 위치) FROM DUAL;
13날짜 함수 / 형변환

날짜 연산 단위(일), TO_CHAR·TO_DATE 형식 마스크, 묵시적 형변환을 함께 정리.

  • SYSDATE: 현재 날짜·시간. SYSTIMESTAMP: 밀리초 포함
  • 날짜 ± 숫자: 일(day) 단위. SYSDATE+1 = 내일. SYSDATE+1/24 = 1시간 후
  • 날짜 - 날짜 = 일수 차이 (소수 포함 가능)
  • MONTHS_BETWEEN(d1, d2): 월 수 차이 (소수 반환)
  • ADD_MONTHS(d, n): n개월 후 날짜
  • LAST_DAY(d): 해당 월의 마지막 날
  • NEXT_DAY(d, '요일'): 다음 지정 요일
  • TO_CHAR(d, 'YYYY-MM-DD'): 날짜→문자
  • TO_DATE('20240101', 'YYYYMMDD'): 문자→날짜
  • TO_NUMBER(s): 문자→숫자. 숫자처럼 보이는 문자열에 묵시적 형변환 발생 가능
  • 묵시적 형변환: 조건에 타입 불일치 시 DB가 자동 변환 — 인덱스 사용 불가 등 부작용 주의
SELECT TO_CHAR(SYSDATE, 'YYYY"년" MM"월" DD"일" HH24:MI:SS') FROM DUAL; -- 2024년 04월 22일 15:30:00 SELECT ADD_MONTHS(TO_DATE('2024-01-31','YYYY-MM-DD'), 1) FROM DUAL; -- 2024-02-29 (윤년 고려하여 말일 처리)
14DECODE / CASE

DECODE는 오라클 전용 동등 비교. CASE는 표준 SQL 범용 조건 분기.

  • DECODE(expr, s1, r1, s2, r2, ..., default): expr = s1이면 r1, = s2이면 r2, 아니면 default
  • DECODE는 = 비교만 가능. >, < 등 범위 비교는 CASE 사용
  • DECODE(NULL, NULL, 'Y', 'N') = 'Y' — NULL끼리 비교는 동등으로 처리 (예외)
  • Simple CASE: CASE col WHEN v1 THEN r1 WHEN v2 THEN r2 ELSE default END
  • Searched CASE: CASE WHEN 조건1 THEN r1 WHEN 조건2 THEN r2 ELSE default END
  • ELSE 없으면 매칭 없을 때 NULL 반환
  • CASE는 SELECT, WHERE, ORDER BY, GROUP BY 어디서나 사용 가능
-- Searched CASE (범위 비교 가능) SELECT name, CASE WHEN score >= 90 THEN 'A' WHEN score >= 80 THEN 'B' WHEN score >= 70 THEN 'C' ELSE 'F' END AS grade FROM students;
15집계 함수

NULL 처리와 COUNT(*) vs COUNT(col) 차이가 핵심 빈출.

  • COUNT(*): NULL 포함 전체 행 수
  • COUNT(col): NULL 제외한 행 수
  • COUNT(DISTINCT col): NULL 제외 + 중복 제거 후 행 수
  • SUM, AVG, MAX, MIN: NULL 자동 제외
  • AVG: NULL 제외 후 합/건수 — 0이 아닌 NULL 제외임을 주의
  • 집계 함수는 WHERE 절에 직접 사용 불가 → HAVING에서 사용
  • 그룹 없이 집계 함수만 쓰면 전체를 1그룹으로 처리 → 결과 1행
  • MAX/MIN은 문자형·날짜형에도 사용 가능
-- 데이터: (1,A), (2,NULL), (3,A), (4,B) SELECT COUNT(*) -- 4 ,COUNT(col2) -- 3 (NULL 제외) ,COUNT(DISTINCT col2) -- 2 (A,B) FROM t;
16GROUP BY / HAVING

그룹핑 전후 필터 위치 구분. SELECT에 집계 아닌 컬럼 포함 여부 주의.

  • WHERE: 그룹핑 전 행 필터 → 집계 함수 사용 불가
  • HAVING: 그룹핑 후 그룹 필터 → 집계 함수 사용 가능
  • GROUP BY에 명시된 컬럼만 SELECT에 집계 없이 사용 가능 (표준)
  • HAVING에서 집계 함수 없이 상수 조건만 쓰는 것도 가능하지만, WHERE가 성능상 유리
  • GROUP BY 결과 행 수 = 그룹 키 유니크 값 조합 수 (NULL도 1개 그룹)
  • GROUP BY에서 SELECT 별칭 사용은 표준 불가 (일부 DBMS 허용)
SELECT department, COUNT(*) AS cnt, AVG(salary) FROM employees WHERE hire_date >= DATE '2020-01-01' -- 그룹핑 전 필터 GROUP BY department HAVING COUNT(*) >= 3 -- 그룹핑 후 필터 ORDER BY cnt DESC;
17JOIN

조인 종류별 결과 행 수와 NULL 생성 위치를 반드시 구분한다.

  • INNER JOIN: 양쪽 테이블 모두 일치하는 행만 반환
  • LEFT OUTER JOIN: 왼쪽 테이블 전체 + 오른쪽 매칭 (없으면 NULL)
  • RIGHT OUTER JOIN: 오른쪽 전체 + 왼쪽 매칭 (없으면 NULL)
  • FULL OUTER JOIN: 양쪽 전체. 매칭 없는 쪽에 NULL
  • CROSS JOIN: 카티션 곱. 결과 행 수 = m × n
  • NATURAL JOIN: 같은 이름의 모든 컬럼으로 자동 조인 (중복 컬럼 1개 출력)
  • SELF JOIN: 같은 테이블을 별칭으로 2번 조인 — 계층 구조 표현에 사용
  • 등가조인: 조인 조건이 = (같다). 비등가조인: 조건이 범위·BETWEEN 등
  • WHERE 조건에 OUTER JOIN 대상 컬럼 필터 추가 시 INNER JOIN과 동일 결과 될 수 있음

18서브쿼리

위치·반환형·상관 여부로 분류. 다중행 연산자와 세트로 외운다.

  • 스칼라 서브쿼리: SELECT 절에 위치. 반드시 1행 1열 반환
  • 인라인 뷰: FROM 절에 위치. 결과를 테이블처럼 사용
  • 중첩 서브쿼리: WHERE / HAVING 절에 위치
  • 단일행 서브쿼리: =, >, < 등 단일행 비교 연산자 사용
  • 다중행 서브쿼리: IN, ANY, ALL, EXISTS 사용
  • 상관 서브쿼리: 외부 쿼리 컬럼을 내부에서 참조. 행마다 반복 실행
  • 서브쿼리에서 ORDER BY 사용 불가 (인라인 뷰는 ROWNUM과 함께 예외)
  • EXISTS vs IN: EXISTS는 존재 여부만 확인 → 대용량에서 EXISTS가 유리한 경우 있음
-- 상관 서브쿼리: 부서 평균 이상 급여를 받는 직원 SELECT name, salary, department FROM employees e1 WHERE salary > (SELECT AVG(salary) FROM employees e2 WHERE e2.department = e1.department); -- 외부 컬럼 참조

19집합 연산자

중복 처리·정렬·컬럼 수 일치 여부가 핵심. 행 수 계산 문제 빈출.

  • UNION: 합집합 + 중복 제거 + 정렬 발생 (ORDER BY 없어도 내부 정렬)
  • UNION ALL: 합집합 + 중복 유지 → 빠름

  • 컬럼명은 첫 번째 SELECT 기준으로 결정됨
  • ORDER BY는 전체 집합 연산 결과에 한 번만 적용 (마지막에 작성)
  • UNION과 UNION ALL 행 수 비교: 중복 행 개수만큼 차이

21DML

INSERT / UPDATE / DELETE 문법과 트랜잭션·COMMIT 전 가시성 주의.

  • INSERT INTO t (col1, col2) VALUES (v1, v2)
  • INSERT INTO t SELECT ...: 서브쿼리 결과 일괄 삽입
  • UPDATE t SET col=값 WHERE 조건: WHERE 생략 시 전체 행 변경 주의
  • DELETE FROM t WHERE 조건: WHERE 생략 시 전체 삭제
  • DML 수행 후 COMMIT 전: 본인 세션에는 변경 보임, 타 세션에는 미반영
  • MERGE INTO: INSERT + UPDATE를 조건에 따라 처리 (UPSERT)
  • DML은 실행 즉시 잠금(Lock) 발생 — COMMIT 또는 ROLLBACK으로 해제
-- MERGE (UPSERT) MERGE INTO target t USING source s ON (t.id = s.id) WHEN MATCHED THEN UPDATE SET t.name = s.name WHEN NOT MATCHED THEN INSERT (id, name) VALUES (s.id, s.name);
22DCL

GRANT / REVOKE. 시스템 권한 vs 객체 권한. WITH GRANT OPTION 연쇄 효과.

  • GRANT 권한 ON 객체 TO 사용자
  • REVOKE 권한 ON 객체 FROM 사용자
  • 시스템 권한: CREATE TABLE, CREATE SESSION 등 DB 수준 권한
  • 객체 권한: SELECT, INSERT, UPDATE, DELETE ON 특정 테이블
  • WITH GRANT OPTION: 받은 권한을 다른 사용자에게 다시 부여 가능
  • WITH GRANT OPTION으로 부여된 권한을 REVOKE 시 연쇄 회수(CASCADE) 발생
  • ROLE: 권한 묶음. 사용자에게 ROLE을 부여해 관리 편의성 향상
  • PUBLIC: 모든 사용자에게 동시 부여
-- A가 B에게 SELECT 부여하고, B가 C에게 재부여 GRANT SELECT ON emp TO B WITH GRANT OPTION; -- A→B GRANT SELECT ON emp TO C; -- B→C REVOKE SELECT ON emp FROM B; -- B와 C 모두 권한 회수됨
23TCL

COMMIT / ROLLBACK / SAVEPOINT. DDL의 암묵적 COMMIT 포인트가 빈출.

  • COMMIT: 트랜잭션 확정. 이후 일반 ROLLBACK 불가
  • ROLLBACK: 트랜잭션 취소. 마지막 COMMIT 시점으로 복구
  • SAVEPOINT sp1: 중간 복구 지점 설정
  • ROLLBACK TO sp1: sp1 이후 변경만 취소 (sp1 이전은 유지)
  • DDL(CREATE, ALTER, TRUNCATE 등) 실행 시 자동(암묵적) COMMIT
  • 정상 세션 종료 시 자동 COMMIT, 비정상 종료 시 자동 ROLLBACK
  • SAVEPOINT는 여러 개 설정 가능. 같은 이름 재사용 시 이전 것 덮어씀
DML-1; -- 변경 A SAVEPOINT sp1; DML-2; -- 변경 B ROLLBACK TO sp1; -- 변경 B만 취소 (변경 A는 유지) COMMIT; -- 변경 A 확정
24제약조건

PK·FK·UNIQUE·NOT NULL·CHECK의 속성 차이를 표로 정리한다.

  • PRIMARY KEY: NOT NULL + UNIQUE. 테이블당 1개만. 자동 인덱스 생성
  • UNIQUE: 중복 불가. NULL은 허용 (오라클은 NULL 여러 개 허용)
  • NOT NULL: NULL 입력 불가
  • CHECK: 입력 값 조건 제한. CHECK (salary > 0)
  • FOREIGN KEY: 참조 테이블의 PK·UNIQUE 컬럼 참조. 참조 무결성 보장
  • FK ON DELETE CASCADE: 부모 행 삭제 시 자식 행도 자동 삭제
  • FK ON DELETE SET NULL: 부모 행 삭제 시 자식 FK 값 NULL로 변경
  • 제약조건 비활성화: DISABLE / ENABLE (오라클)

25VIEW

논리적 가상 테이블. 보안·단순화 목적. 갱신 불가 조건을 외운다.

  • 뷰는 쿼리를 저장한 객체 — 실제 데이터 저장 공간 없음
  • 보안: 민감 컬럼 제외하고 뷰 생성 후 접근 제어
  • 단순화: 복잡한 JOIN·집계를 뷰로 추상화
  • 뷰를 통한 DML이 불가한 경우: GROUP BY, DISTINCT, 집계 함수, ROWNUM, UNION 포함 시
  • 단순 뷰(단일 테이블 + 집계 없음): DML 대체로 가능
  • WITH CHECK OPTION: 뷰 조건을 위반하는 DML 차단
  • WITH READ ONLY: 뷰를 통한 모든 DML 차단
  • MATERIALIZED VIEW: 실제 데이터를 물리적으로 저장하는 뷰 — 조회 성능 향상
CREATE OR REPLACE VIEW emp_dept_view AS SELECT e.name, e.salary, d.department_name FROM employees e JOIN departments d ON e.dept_id = d.id WHERE e.salary > 3000 WITH CHECK OPTION; -- salary <= 3000 UPDATE 차단
26ROLLUP / CUBE / GROUPING SETS / GROUPING

소계·총계 행 수 계산이 빈출. 각 함수가 만드는 그룹 조합을 정확히 파악한다.

  • ROLLUP(a, b): (a,b), (a), () — n+1 개 조합 (계층적 소계)
  • CUBE(a, b): (a,b), (a), (b), () — 2ⁿ 개 조합 (모든 조합)
  • GROUPING SETS((a,b),(a),()): 명시한 조합만 생성
  • GROUPING(col): 소계·총계 행에서 해당 컬럼이 집계에 사용되면 1, 아니면 0
  • 소계 행의 해당 컬럼 값은 NULL로 표시 → GROUPING으로 NULL과 구분
  • ROLLUP 컬럼 수 n → 생성 그룹 수 n+1
  • CUBE 컬럼 수 n → 생성 그룹 수 2ⁿ
-- ROLLUP(dept, job): 3가지 그룹 생성 -- (dept, job), (dept), () → 소계 + 총계 SELECT dept, job, SUM(salary) FROM emp GROUP BY ROLLUP(dept, job); -- GROUPING으로 총계 행 식별 SELECT dept, CASE GROUPING(dept) WHEN 1 THEN '합계' ELSE dept END AS dept_label, SUM(salary) FROM emp GROUP BY ROLLUP(dept);
27윈도우 함수 (ROWS/RANGE, RANK/DENSE_RANK/ROW_NUMBER)

RANK 계열 3종 차이와 ROWS·RANGE 프레임 기준을 정확히 구분한다.

  • RANK: 동순위 허용. 다음 순위는 건수만큼 건너뜀 (1,2,2,4)
  • DENSE_RANK: 동순위 허용. 다음 순위는 바로 다음 (1,2,2,3)
  • ROW_NUMBER: 동순위 없이 순서대로 고유 번호 (1,2,3,4)
  • ROWS BETWEEN: 물리적 행 수 기준 프레임
  • RANGE BETWEEN: 정렬 값 기준 논리적 프레임 (같은 값 행 묶음)
  • ROWS UNBOUNDED PRECEDING: 파티션 첫 행부터 현재 행까지
  • CURRENT ROW: 현재 행. UNBOUNDED FOLLOWING: 파티션 마지막 행까지
  • 집계 함수와 달리 원본 행을 유지한 채 결과를 추가 컬럼으로 붙임
SELECT name, salary, RANK() OVER (ORDER BY salary DESC) AS rnk, DENSE_RANK() OVER (ORDER BY salary DESC) AS d_rnk, ROW_NUMBER() OVER (ORDER BY salary DESC) AS rn, SUM(salary) OVER (PARTITION BY dept ORDER BY hire_date ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW) AS running_sum FROM employees;
28계층형 쿼리 기초

START WITH + CONNECT BY PRIOR. PRIOR 위치가 방향을 결정한다.

  • START WITH: 계층의 루트 행 조건
  • CONNECT BY PRIOR 자식컬럼 = 부모컬럼: 부모→자식 방향 (하향)
  • CONNECT BY PRIOR 부모컬럼 = 자식컬럼: 자식→부모 방향 (상향)
  • LEVEL: 루트가 1. 한 단계 내려갈수록 +1
  • CONNECT_BY_ISLEAF: 리프 노드면 1, 아니면 0
  • ORDER SIBLINGS BY: 계층 구조를 유지하면서 형제 노드 간 정렬
  • CONNECT_BY_ROOT col: 루트 행의 col 값 반환
  • SYS_CONNECT_BY_PATH(col, '/'): 루트부터 현재까지 경로 문자열
SELECT LPAD(' ', (LEVEL-1)*2) || emp_name AS tree, LEVEL, CONNECT_BY_ISLEAF AS is_leaf FROM employees START WITH manager_id IS NULL -- 최상위 직원(루트) CONNECT BY PRIOR emp_id = manager_id -- 부모→자식 방향 ORDER SIBLINGS BY emp_name;

Section 4. 데이터 모델링 핵심 (12개 주제)

29엔터티

업무상 관리 필요한 대상. 식별 가능성·인스턴스 2건 이상이 성립 조건.

  • 엔터티 정의 조건: ① 업무에 필요한 정보 ② 식별 가능 ③ 인스턴스 2건 이상 존재 ④ 속성 2개 이상 ⑤ 관계 존재
  • 유형 엔터티: 실체가 있음 (고객, 직원, 제품)
  • 개념 엔터티: 물리적 형태 없음 (조직, 보험상품)
  • 사건 엔터티: 업무 수행으로 발생 (주문, 이력, 청구)
  • 기본 엔터티: 독립적으로 생성 가능. 중심 엔터티: 기본 엔터티로부터 발생. 행위 엔터티: 2개 이상 관계로 발생
  • 한 엔터티는 반드시 하나 이상의 식별자(PK)를 가져야 함
30속성 분류

기본·설계·파생 속성의 구분과 파생 속성 중복 저장의 문제점을 이해한다.

  • 기본 속성: 업무로부터 직접 수집 (이름, 가격, 날짜)
  • 설계 속성: 시스템 운영을 위해 인위적으로 추가 (코드, 일련번호)
  • 파생 속성: 다른 속성으로부터 계산·가공 (나이←생년월일, 합계←단가×수량)
  • 파생 속성은 중복 저장이므로 데이터 정합성 관리 비용 증가
  • 단일값 속성 vs 다중값 속성: 다중값은 별도 엔터티로 분리 권장
  • 속성도 도메인을 가지며 허용 값 범위를 지정할 수 있음
31도메인

속성의 허용 값 범위와 규칙. 자료형보다 포괄적인 개념.

  • 도메인 = 속성이 가질 수 있는 값의 범위 + 형식 + 제약
  • 예: '성별' 속성의 도메인 = {'M', 'F'} 또는 {0, 1}
  • 도메인 정의는 데이터 품질 관리의 출발점
  • 같은 도메인을 공유하는 속성은 비교·조인이 의미 있음
  • 자료형(VARCHAR2, NUMBER)은 도메인의 일부 — 형식 제약에 해당
  • 코드 도메인: 코드 테이블로 관리하여 일관성 확보
32제약조건 (모델링 관점)

업무 규칙을 데이터 모델에 표현. DB 제약조건과 연결하여 이해한다.

  • 엔터티 무결성: PK는 NULL·중복 불가
  • 참조 무결성: FK는 참조 대상 PK 값 또는 NULL만 허용
  • 도메인 무결성: 속성 값이 정의된 도메인 내 값이어야 함
  • 사용자 정의 무결성: 업무 규칙에 의한 제약 (CHECK 등)
  • NULL 허용 여부 = 속성의 필수/선택 여부와 직결
  • 모델링 단계에서 업무 제약을 명확히 표현해야 구현 시 누락 없음
33관계 / IE / Barker 표기법

카디널리티와 필수/선택 참여를 표기법별로 읽는 능력이 필요하다.

  • 관계 = 엔터티 간의 업무적 연관. 관계명·관계차수·선택성으로 정의
  • 1:1: 한 인스턴스가 반대편 정확히 1개와 대응
  • 1:M: 한 인스턴스가 반대편 0~N개와 대응
  • M:N: 양쪽이 서로 다대다. 구현 시 교차 테이블로 해소
  • IE 표기: 선 끝에 까마귀발(다수), 수직선(1), 원(0/선택) 사용
  • Barker 표기: 실선(필수), 점선(선택), 수직선(1), 까마귀발(다수)
  • 필수 참여: 반드시 상대방 인스턴스 존재. 선택 참여: 없어도 됨
34식별자

주식별자 4대 조건(유일성·최소성·불변성·존재성)과 식별자 분류를 외운다.

  • 주식별자 조건: 유일성·최소성·불변성·존재성(Not Null)
  • 내부 식별자: 엔터티 내부에서 스스로 생성 (주문번호)
  • 외부 식별자: 타 엔터티 관계로부터 가져온 식별자 (FK)
  • 단일 식별자: 속성 1개로 식별. 복합 식별자: 속성 2개 이상
  • 본질 식별자: 업무 자체 의미를 가진 식별자. 인조 식별자: 편의를 위해 인위적으로 생성 (자동채번)
  • 보조 식별자: 유일성은 있으나 PK로 사용하지 않는 식별자 (UNIQUE)
  • 식별자는 시간이 지나도 변하지 않아야 함(불변성) — 자주 변하는 값은 PK 부적합
35식별 / 비식별 관계

부모 키가 자식 PK에 포함되는지 여부가 핵심 구분 기준.

  • 식별 관계: 부모의 PK가 자식의 PK 구성요소로 전이됨 → 실선으로 표기
  • 비식별 관계: 부모의 PK가 자식의 일반 FK(비PK)로 존재 → 점선으로 표기
  • 식별 관계: 부모 없이 자식 독립 존재 불가 (강한 연결)
  • 비식별 관계: 자식이 독립 존재 가능 (유연성 높음)
  • 식별 관계가 여러 단계 반복되면 PK 컬럼이 점점 많아져 복잡도 증가
  • 실무에서는 비식별 관계 + 인조 식별자 조합이 많이 사용됨
구분식별 관계비식별 관계
부모 PK 위치자식 PK 포함자식 일반 컬럼(FK)
자식 독립 존재불가가능
표기(IE)실선점선
36ERD 그리기

업무 문장 → 엔터티·관계 추출 → PK/FK 배치 → M:N 해소 순서로 접근.

  • 엔터티 추출: 명사 → 인스턴스가 2건 이상 존재할 수 있는지 확인
  • 관계 추출: 동사 → 방향성과 카디널리티 결정
  • M:N 관계는 반드시 교차(Association) 엔터티로 해소
  • PK 컬럼 → FK로 이어지는 방향을 화살표로 시각화
  • 필수/선택 참여 여부를 선 표기에 반영
  • 식별/비식별 관계를 실선/점선으로 구분
  • ERD는 개념 → 논리 → 물리 모델링 단계로 발전
37PIVOT / UNPIVOT

행↔열 변환 구조. PIVOT은 행 값을 열로 펼치고, UNPIVOT은 반대.

  • PIVOT: 집계 함수 + FOR 컬럼 IN (값 목록) → 행 값을 열 헤더로
  • UNPIVOT: 여러 컬럼을 하나의 컬럼 값으로 내림
  • PIVOT 결과 컬럼 수 = IN 목록 값 개수 (정적 PIVOT)
  • 동적 PIVOT: IN 목록을 동적으로 구성 — 보통 동적 SQL 필요
  • PIVOT 없이 CASE WHEN으로도 동일 결과 구현 가능 → 시험에서 자주 비교 출제
-- PIVOT 예시: 분기별 매출 집계 SELECT * FROM ( SELECT year, quarter, sales FROM revenue ) PIVOT (SUM(sales) FOR quarter IN (1 AS Q1, 2 AS Q2, 3 AS Q3, 4 AS Q4));
38정규화

함수 종속성 제거 단계별 정리. 1NF~3NF를 종속성 관점으로 이해한다.

  • 1NF: 모든 속성이 원자값 (반복 그룹·다중값 제거)
  • 2NF: 1NF + 부분 함수 종속 제거 (복합 PK의 일부에만 종속된 속성 분리)
  • 3NF: 2NF + 이행 함수 종속 제거 (PK→A→B 관계에서 A→B 분리)
  • BCNF: 모든 결정자가 후보키여야 함 (3NF 강화)
  • 정규화 목적: 데이터 중복 제거, 이상 현상 방지, 무결성 향상
  • 정규화가 높을수록 조인이 많아져 조회 성능 저하 가능 → 반정규화 고려
  • 함수 종속: X → Y (X값이 결정되면 Y값도 결정됨)
정규형조건제거 대상

39이상 현상

삽입·갱신·삭제 이상 3가지를 사례 중심으로 외운다. 비정규화 구조의 부작용.

  • 삽입 이상: 원하는 데이터 입력 시 불필요한 값도 함께 입력해야 함
  • 갱신 이상: 중복 데이터 중 일부만 수정 시 불일치 발생
  • 삭제 이상: 특정 데이터 삭제 시 필요한 다른 정보도 함께 삭제됨
  • 세 이상 현상 모두 데이터 중복(비정규화)에서 비롯됨
  • 정규화를 통해 이상 현상 방지 → 테이블 분리
  • 예: 학생-과목-담당교수 한 테이블에 저장 시 과목만 삭제하면 담당교수 정보 소실 (삭제 이상)
40반정규화

성능을 위해 의도적 중복 허용. 언제, 무엇을 반정규화하는지 판단 기준이 중요.

  • 반정규화: 정규화 테이블을 성능 이유로 다시 통합하거나 중복 추가
  • 조회 성능 향상이 목적 — 잦은 조인 감소
  • 단점: 데이터 정합성 관리 비용 증가, DML 성능 저하 가능
  • 반정규화 기법: 테이블 병합, 중복 컬럼 추가, 집계 테이블 추가, 이력 테이블 분리
  • 적용 시점: 정규화 완료 → 성능 테스트 → 문제 지점만 선택적 적용
  • 중복 컬럼 추가: 자주 JOIN하는 컬럼을 미리 복사 저장
  • 집계 테이블: SUM, COUNT 등 결과를 별도 테이블에 주기적 저장

41윈도우 함수 기본 구조 / 핵심 함수

OVER 절 구조 해석. LAG/LEAD/SUM OVER 등 분석 함수를 집계와 구분한다.

  • 기본 구조: 함수() OVER ([PARTITION BY] [ORDER BY] [프레임절])
  • PARTITION BY: 윈도우 내 그룹 구분 (GROUP BY와 달리 행 유지)
  • ORDER BY: 윈도우 내 순서 정의 (프레임 기준)
  • LAG(col, n): 현재 행 기준 n행 앞 값. n 생략 시 1
  • LEAD(col, n): 현재 행 기준 n행 뒤 값
  • LAG/LEAD 대상 없으면 기본값 NULL (세 번째 인수로 기본값 지정 가능)
  • FIRST_VALUE(col) / LAST_VALUE(col): 윈도우 첫/마지막 값
  • NTILE(n): 행을 n개 버킷으로 균등 분할. 버킷 번호 반환
  • 집계 함수(SUM, AVG)도 OVER 절과 함께 사용 시 분석 함수로 동작 — 원본 행 유지
SELECT name, salary, hire_date, LAG(salary, 1, 0) OVER (PARTITION BY dept ORDER BY hire_date) AS prev_salary, LEAD(salary) OVER (PARTITION BY dept ORDER BY hire_date) AS next_salary, NTILE(4) OVER (ORDER BY salary DESC) AS quartile FROM employees;
42정규표현식

패턴 해석형 문제 중심. 주요 메타문자와 오라클 REGEXP 함수를 정리한다.

  • ^: 문자열 시작. $: 문자열 끝
  • .: 임의의 한 문자. \d: 숫자, \w: 단어문자
  • *: 0회 이상. +: 1회 이상. ?: 0 또는 1회
  • {n}: 정확히 n회. {n,m}: n~m회
  • [abc]: a,b,c 중 하나. [^abc]: a,b,c 제외
  • [A-Z]: 대문자 범위. [0-9]: 숫자 범위
  • REGEXP_LIKE(col, 패턴): 패턴 일치 여부 (WHERE 조건)
  • REGEXP_SUBSTR: 패턴 일치 부분 문자열 추출
  • REGEXP_REPLACE: 패턴 일치 부분 치환
  • LIKE와의 차이: LIKE는 %, _ 두 가지 패턴만 → 정규표현식이 훨씬 강력
-- 이메일 형식 검증 SELECT name FROM users WHERE REGEXP_LIKE(email, '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$'); -- 숫자로만 이루어진 전화번호 WHERE REGEXP_LIKE(phone, '^[0-9]{10,11}$');
43트랜잭션

ACID 4가지 속성과 격리 수준별 읽기 이상 현상을 함께 정리한다.

  • 원자성(Atomicity): 트랜잭션 내 작업은 전부 완료 or 전부 취소
  • 일관성(Consistency): 트랜잭션 완료 후 DB가 일관된 상태 유지
  • 고립성(Isolation): 동시 실행 트랜잭션은 서로 영향 차단
  • 지속성(Durability): COMMIT된 데이터는 장애 후에도 유지
  • Dirty Read: COMMIT 안 된 데이터를 다른 트랜잭션이 읽음
  • Non-Repeatable Read: 같은 쿼리를 두 번 실행 시 결과 다름 (UPDATE)
  • Phantom Read: 같은 범위 조회 시 행 수가 달라짐 (INSERT)
  • 격리 수준 높을수록 이상 현상 감소, 동시성 저하

44계층형 쿼리 심화

PRIOR 방향·LEVEL·리프 여부·형제 정렬. 결과 레벨 해석이 최종 관문.

  • PRIOR 위치 완전 정리: CONNECT BY PRIOR A = B → A가 자식, B가 부모
  • 하향식 (루트→리프): CONNECT BY PRIOR 자식_id = 부모_id
  • 상향식 (리프→루트): CONNECT BY PRIOR 부모_id = 자식_id
  • LEVEL: 루트=1, 루트의 직속 자식=2, ...
  • CONNECT_BY_ISLEAF = 1: 해당 행이 리프 노드 (자식 없음)
  • ORDER SIBLINGS BY col: 계층 순서 유지하면서 같은 부모의 자식들끼리 정렬
  • CONNECT_BY_ROOT col: 루트 노드의 col 값 반환 (오라클)
  • SYS_CONNECT_BY_PATH(col, sep): 루트~현재 노드까지 경로 문자열 반환
  • 무한 루프 방지: NOCYCLE 옵션 + CONNECT_BY_ISCYCLE 의사 컬럼 사용
-- 심화: 경로 + 루트 + 리프 조회 SELECT SYS_CONNECT_BY_PATH(emp_name, '/') AS path, CONNECT_BY_ROOT emp_name AS root_name, LEVEL, CONNECT_BY_ISLEAF AS is_leaf FROM employees START WITH manager_id IS NULL CONNECT BY NOCYCLE PRIOR emp_id = manager_id ORDER SIBLINGS BY emp_name;
